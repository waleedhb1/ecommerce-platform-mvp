نظرة عامة على المشروع

متجر نواة هو نموذج أولي (MVP) لمنصة إدارة متجر إلكتروني باللغة العربية واتجاه RTL. يعمل بالكامل في المتصفح بدون خادم خلفي — جميع البيانات وهمية وتُحمَّل عبر واجهة برمجية محاكية (Mock API).

الميزات الرئيسية

  • لوحة مبيعات تفاعلية مع إحصائيات ورسوم بيانية
  • إدارة المنتجات مع شارات المخزون (متوفر / منخفض / نفد)
  • متابعة الطلبات مع فلترة وتحديث الحالة ونافذة تفاصيل
  • قاعدة عملاء مع تصنيفات (عادي، فضي، ذهبي، بلاتيني)
  • إدارة الشحن والتسويق (كوبونات وحملات)
  • إعدادات المتجر وبوابات الدفع (مدى، Apple Pay، تابي) ومناطق الشحن

تقنيات HTML5 + CSS3 + JavaScript خام (Vanilla JS) — بدون إطار عمل أو مكتبات خارجية.

دوال API المتاحة

الدالةالوصفالمعاملات
getDashboard()إحصائيات لوحة المبيعات والرسم البياني—
getProducts(filter, q)قائمة المنتجاتall | active | low | out | draft + بحث
getOrders(filter, q)قائمة الطلباتall | new | processing | shipped | delivered | cancelled + بحث
updateOrderStatus(id, status)تحديث حالة طلبمعرّف الطلب + الحالة الجديدة
getCustomers(q)قائمة العملاءنص بحث اختياري
getShipping()طرق الشحن وإحصائيات التسليم—
getMarketing()الكوبونات والحملات—
getSettings()إعدادات المتجر والدفع والشحن—

كل دالة تستخدم delay() لمحاكاة زمن استجابة الشبكة (220ms افتراضيًا، 350ms للتحديثات).

التكامل مع خادم خلفي مستقبلي

الهيكل الحالي مصمم ليسهل استبدال Mock API بخادم حقيقي:

الخطوة 1 — استبدال mock-api.js

أنشئ ملف api.js يحافظ على نفس واجهة الدوال لكن يستدعي REST API:

const EcommerceAPI = (async () => {
  const BASE = "https://api.nawa-store.sa/v1";

  async function fetchJSON(path) {
    const res = await fetch(`${BASE}${path}`, {
      headers: { Authorization: `Bearer ${token}` }
    });
    return res.json();
  }

  return {
    async getDashboard() { return fetchJSON("/dashboard"); },
    async getProducts(f, q) { return fetchJSON(`/products?filter=${f}&q=${q}`); },
    // ... باقي الدوال بنفس النمط
  };
})();

الخطوة 2 — نقاط النهاية المقترحة (REST)

الطريقةالمسارالوصف
GET/api/dashboardإحصائيات لوحة المبيعات
GET/api/productsقائمة المنتجات مع فلترة
POST/api/productsإضافة منتج
GET/api/ordersقائمة الطلبات
PATCH/api/orders/:id/statusتحديث حالة الطلب
GET/api/customersقائمة العملاء
GET/api/shippingطرق الشحن
GET/api/marketingكوبونات وحملات
GET/PATCH/api/settingsإعدادات المتجر

الخطوة 3 — اعتبارات الإنتاج

  • المصادقة: أضف JWT أو OAuth2 — حاليًا لا يوجد نظام تسجيل دخول
  • قاعدة البيانات: PostgreSQL أو MySQL مع جداول products, orders, order_items, customers
  • بوابات الدفع: تكامل مع HyperPay أو Moyasar لمدى و Apple Pay و Tabby
  • الشحن: تكامل API مع أرامكس وسمسا لتتبع الشحنات
  • العملة: SAR مع دعم ضريبة القيمة المضافة 15%
  • RTL: الحفاظ على dir="rtl" وخط عربي في الواجهة

نصيحة لا تحتاج لتعديل app.js عند الانتقال للخادم — طالما حافظت api.js على نفس أسماء الدوال وشكل البيانات المُرجَعة.

مكوّنات الواجهة (CSS)

ملف styles.css يتضمن جميع المكوّنات التالية الجاهزة للاستخدام:

  • grid-stats — شبكة بطاقات إحصائية (4 أعمدة، متجاوبة)
  • card / stat-card — بطاقات المحتوى والإحصائيات
  • table / table-wrap — جداول بيانات مع تمرير
  • tag — شارات الحالة (success, warn, info, danger, purple, neutral)
  • stock-badge — شارات المخزون (in, low, out)
  • pipeline — مسار حالة الطلب (4 مراحل)
  • bar-chart — رسم بياني عمودي
  • modal / toast — نوافذ منبثقة وإشعارات
  • gateway-badge — شارات بوابات الدفع
  • responsive — نقاط توقف 1200px و 900px
